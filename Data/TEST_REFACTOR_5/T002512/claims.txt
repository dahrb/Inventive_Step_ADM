“Systems and Method for Proxying Data Access Commands in a Storage System
Cluster”

New Claims i to 17

1. A method for proxying data access commands from a first storage system
(200A) to a second storage system (200B) in a storage system cluster (130),
comprising :

configuring a proxy port on the first storage system (200A),

wherein the proxy port of the first storage system (200A) is utilized for proxying data
access to the second storage system (200B);

receiving a data access command at the proxy port on the first storage system
(200A),

wherein the data access command is directed to the second storage system (200B)
Cr],

wherein the data access command includes a world wide port name (WWPN) and a
logical unit number (lun) identifier;

generating a file-level data access request from the received data access command by
identifying a file handle associated with a vdisk on the second storage system (200B)
using the WWPN and lun identifier,

wherein the file handle is a temporary reference assigned by an operating system of
the first storage system (200A) to the vdisk used through a session to access the
vdisk ;

forwarding the file-level data access request over a cluster interconnect (110) to the
second storage system (200B) ;
